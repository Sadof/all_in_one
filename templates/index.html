Приветствую на сайте с моими наработками, сделанными с помощью Bootstrap и Django

Весь бекэнд соответсвенно написан на django с минимальным использованием дополнительных модулей. Все графическое оформление выполнено с целью минимального визуального представления, не уделяя этому достаточного внимания.

Модули:

  • Блог:

    Первый сделанный модуль. В ознакомительных целях доступ к созданию тестов доступен каждому зарегестрированному пользователю(так должен быть доступен пользователям с соответсвующими правами).

    Редактирование и удаление постов доступно автору и стаффу.

    В каждом детальном просмотре поста имеется возможность оставлять коменнтарий(с возможность его редактировать и удалять ajax'ом для оставившего комментарий и удаления для стаффа).

    В правом нижнем углу имеется "Админ панель" для возможности быстрого перехода для создания, редактированияи удаления(должно быть видимо для определенной группы пользователей).

    Также есть возможность создавать тэги и добавлять их к постам для поиска по ним.

    • Create post Создание постов с 3 полями: название(обязательное поле), сланг(опциональный) при отсутсвуется слагается название, и текст поста. Также возможность добавить таг из существующихю
    • Post list Список всех созданных постов.
    • "blog/слаг поста/" Детальный просмотр поста. Возможность добавления комментариев и реплаев имеющиеся комментарии для зарегестрированный пользователей. Возможно редактирование и удаления комметариев.
    • "blog/слаг поста/edit/" Редактирование поста с соответсвующим слагом. Это возмжность в данном случаем может быть некоректна, т.к. пост сразу попадает в общий доступ и редактирование в таком случае не лучшая идея :). Доступно для автора.
    • "blog/слаг поста/delete/" Удаление поста. Доступно для автора и стаффа.
    • Tag list Список всех тэгов. При нажатии на какой либо тэг выведется список постов с данным тэгом.
    • Tag add Добавление тэгов.

  • Tests

    Модуль с возможность создавать пользовательские тесты, публиковать и выполнять их.

    • Create test Создание основы тесы( название, слаг и описание), также как и в создании поста, slug является оптициональным полем автогенерируемым из названия. Доступно для авторизованных пользоветелейю
    • "test/edit/id теста/" Главная страница редактирования тестов до их публикация( после нее такая возможность пропадает). На ней редактируется основаная информация о тесте.

      • В левом боковом меню есть возможности:

      • "test/edit/id теста/preview/" предпросмотра визуального вида теста.
      • Список всех страниц данного теста с возможностью поменять их порядок перетаскиванием. С переход к редактированию конкретной страницы.
      • "test/edit/id теста/add_page/" добавление основы страницы вопроса: названия страницы(необязательное поле, картинка(необязательное), и сам вопрос.
      • "test/edit/id теста/порядок вопроса/" редактирование страницы вопроса, и добавления новых полей таких как: Right answer( выбор правильного ответа из возможных введенных вариантов), далее текст, появляющий при неверном ответе, и объясняющим верный ответ( необязательное поле). И поля с добавлением возможных вариантов ответа на данных вопрос.
      • И публикация теста. Необходимымми на данный момент требованиями для публикации: минимум 5 вопросов, по 2 ответа для каджого вопроса и указание правильного ответа на вопрос. При выполнение условий, тест становится видимым всем и пропадает возмость для редактирования.

    • Test list Содержит список всех опубликованных тестов с их информацией.
    • test/слаг теста/ Выполнение теста. На начальной страницы выводятся: Название теста, его описание. Также топ 3 ваших результата предыдущих выполнений( в данном исполнение тестов это не самая полезная функция, но при добавлении рандомного решафла вопросов, то в этом появляется больше смысла. Добавлен дял использоваия проверки ManyToMany). После ответа на все вопросы выводится количество правильных ответов, предлагается лайкнуть или дизлайкнут тест, и список всех вопросов с выделенными правильными и описание ошибки, если такая имеется.

  • Accounts

    • Login
    • Logout
    • Registration
    • Home page Ссылна на свой профиль, в отличие accounts/некий никнейм/ имееются дополнительные функции по редартиолапнию профиля и доп информации о собственном контенте. Так же на странице имеется список, сделанных пользоветем постов и тестов по 5 штук, и ссылки на полный список.
    • Редактирование данных профиля.
    • Список тестов Полный список собственных тестов с их инофрмацией и ссылка на изменение, удаление, просмотр.

  • "Api"

    • Test list Выполучает все опубликованные тесы.
    • /api/v1/tests/some id/ Детальный ввывод отделного опубликованного теста.
    • /api/v1/tests/some id/edit/ Редактор неопубликованного теста автором.
